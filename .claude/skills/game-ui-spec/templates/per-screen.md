#### 범례

와이어프레임의 모든 번호를 해설. `UX 의도`·`동작 기준`은 필수.
> 동작 기준은 **측정 가능**("200ms 이내", "1초 내")하게, "빠르게" 금지. 상태에 따라 갈리는 요소
> (보유 가득/첫 발생/협동 vs 솔로)는 **각 경우를 따로** 적는다. 수치 임계값/보장("HP55% 페이즈",
> "Rare 1개 보장")은 **GDD 줄을 인용하거나 `[확정 필요 — GDD]`로 표시**한다. `UX 의도`는 프레임워크
> 이름 없이 평이하게(예: "7시까지 남은 시간을 한순간도 못 가리게").

#### 상태 매트릭스

비기본 상태는 마이크로카피+시각표시(아이콘+색+텍스트)+복구법. 색만으로 의미 전달 금지. `loading`은 **진입 애니메이션과 데이터 미수신(예: 협동 원격 플레이어 HP 지연)을 구분**한다. `—`는 빈칸으로 두지 말고 사유를 적는다(예: "N/A: 읽기 전용·동기 로컬값").
