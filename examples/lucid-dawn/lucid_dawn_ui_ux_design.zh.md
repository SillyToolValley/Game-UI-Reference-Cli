## 1. 概述与目标

- **本文解决的问题**：GDD §9-1（画面流转）与 §9-2（HUD 表格）只确定了"什么放在哪里"。要真正落地，还需要每个画面的状态、输入、数据绑定、边界情况、无障碍，以及*为什么* —— 从菜单（局外）一直到战斗/奖励/结算（局内）。
- **核心体验 (GDD §1-1)**："压迫 → 瞬间读招 → 完美闪避 → 清醒爆发 → 净化/成长 → 靠自己醒来"。UI 绝不能遮挡这个循环。
- **可衡量的成功**
  - 局外：新手玩家在 **3 分钟**内开始一局（菜单深度 + 标签清晰度）。
  - 局内：任意时刻玩家都能在 1s 内读出**距 7:00 的剩余时间**(§3-2)；在 180 敌人硬上限 (§6-4) 下，玩家、致命弹幕、危险区域与拾取物仍可通过**颜色 + 形状**区分 (§13-3 #5)。
  - 奖励流程遵循 GDD §12-3：**发放技能点 → 技能树 → 独立的 4 选物**。
  - BM (§10)：无付费战力、无抽卡 → meta 画面**仅靠游玩解锁（无购买 UI）**。
- **范围内**：局外 O1 标题/主菜单、O2 角色选择、O3 关卡/难度、O4 选项、O5 Meta 进度（解锁/图鉴）、O6 排行榜；局内 I1 HUD、I2 HUD（Boss/隐藏）、I3 升级/选物、I4 Boss 奖励（技能树 + 4 选1）、I5 暂停、I6 结算（3 分支）。
- **范围外（本文）**：仅合作相关的 UI（大厅/团队槽/幽灵/联合终极 —— GDD §8）在 VS 中未实现 → 在 §6-1 作为风险追踪；新手教程画面、商店（按 BM 没有）、过场打磨留待后续。

## 2. 用户与情境

- **画像**：survivor-like / 20 分钟生存 / 弹幕地狱 / 动作 roguelite / 合作 PvE 玩家 (GDD §1)。从新手（首次 Dawn Wake）到老手（Dream Break / 无伤记录）并存。
- **平台 / 输入**：优先 PC/Steam。键鼠与手柄皆支持 (GDD §2-5)。触屏超出范围（设计保持可向手柄对等扩展）。
- **进入情境**：局外 = 非战斗、焦点驱动；局内 = 实时战斗，但升级/奖励在暂停/保护下选取 (§4-5, §7-1)。一局 = 6:40→7:00（20 分钟，内部 0–1200s）。HUD 仅在局内显示；菜单/结算中隐藏。VS 阵容 = Kohaku、Toko（其余 4 名锁定）；关卡 = LD-001 (§3, §5, §6-2)。

### 2-1. 设计原则（适用于所有画面）
| # | 原则 | 在此处的含义 | 若被违反 |
| --- | --- | --- | --- |
| P1 | 即时反馈 | 每个操作/状态变化都立刻有回应（声音 + 显示） | 手感/感知薄弱 |
| P2 | 危险可读性 | 玩家 > 即时威胁（弹幕/区域/蓄力）> 拾取物 > 环境 | 看不见危险 |
| P3 | 混乱中可读 | 180 敌人时核心仍靠颜色+形状+声音；限制 juice | 致命招式被淹没 |
| P4 | 一致性 | 跨画面固定位置/颜色/图标/操作/条形；刻意例外要标明 | 持续困惑 |
| P5 | 渐进式披露 | 先核心，其余按需出现（清醒/净化/Boss 在首次出现时教学） | 新手过载 |
| P6 | 无障碍 | 不仅靠颜色（颜色+形状+文字）、字幕、震动/闪光/插画开关、改键 | 色盲/敏感人群被排除 |

## 3. 流程线框（Wireflow）

```text
[标题/主菜单 O1] --任意键/单人--> ┬ [角色 O2] --确认--> [关卡/难度 O3] --开始--> (局内)
                                     ├ [选项 O4] · [Meta O5] · [排行榜 O6] · 退出
(局内，显示 HUD)
[RunStart] --1s--> [NormalRun I1]
   [NormalRun] --升级--> [升级 I3] --确认--> [NormalRun]   (暂停)
   [NormalRun] --ESC--> [暂停 I5] --继续--> [NormalRun]
   [NormalRun] --净化阈值--> [BossWarning] --出现--> [BossFight I2]
   [BossFight] --击败--> [BossReward I4: 技能树→4选1] --> [NormalRun]
   [4 封印破除] --> [HiddenBossAttempt I2]  (Boss HP + 剩余时间)
(结束 → 结算)
[HP0]→FailedWake ┐  [7:00]→DawnWake ├─> [Result I6]   [7:00 前击杀隐藏 Boss]→DreamBreak ┘
```

![流程图](wireframes/flow.svg)

| 转换 | 触发（事件/条件） | 去向 | 状态 ↔ HUD |
| --- | --- | --- | --- |
| 标题 → 主菜单 | 任意键 | 主菜单 | 隐藏 HUD |
| 主菜单 → 角色 → 关卡 → 局内 | 单人 → 确认 → 开始 | 局内 | 显示 HUD |
| NormalRun → 升级 | `RunTelemetry.level_reached` ++ | I3 模态 | 战斗**暂停** |
| NormalRun → 暂停 | ESC/Start | I5 | 战斗**暂停** |
| NormalRun → BossWarning | `BossThresholdReached` (§12-1) | 警告 5s | 保留 HUD |
| BossWarning → BossFight | `BossSpawned` (§12-1) | I2 | 顶部中央 Boss HP 接管 |
| BossFight → BossReward | `BossDefeated` (§12-1) | I4 | 奖励流程，**8s 保护** |
| 4 封印 → 隐藏 Boss | `HiddenBossSpawned` (§12-1) | I2 隐藏 | Boss HP + 剩余时间 |
| 局内 → 结算 | `AlarmReached`/HP0/`DreamBreakAchieved` (§12-1) | I6 | 隐藏 HUD |

#### 导航
单人→O2 / 选项→O4 / 图鉴·解锁→O5 / 排行榜→O6 / 合作→（VS 锁定）/ 退出→确认弹窗。默认焦点 **Single Play**。

#### 边界情况
退出 = 不可逆 → 确认弹窗。无存档（首次启动）：meta/图鉴为空但可进入。合作锁定："敬请期待"，不可启动。

#### 无障碍
焦点 = 颜色+边框+声音；锁定 = 锁+文字。字幕/文字缩放。授权行需可读对比度 (§13-3 #1)。

#### UX 设计意图
- **一眼读懂**：logo→竖向菜单成一线；仅 6 项，新手能立刻找到"Single Play"。
- **操作·防误触**：默认焦点在单人；退出需确认；合作灰显"敬请期待"杜绝无效点击。
- **手感·爽点**：右侧主视觉从第一屏起就传达"可爱噩梦"的基调。
- **首次 vs 重复**：新手走单人；老手直接跳到 meta/排行榜。
- **无障碍**：锁定/选中不只靠颜色；授权行清晰可读。
- **需特别注意（前 3）**：① 官方/授权混淆 (§13-3 #1) → 预留授权说明行（法务另行评审）；② 菜单过载 → 少项 + 默认焦点；③ 合作无效点击 → 锁定标签。

#### 待定问题
VS 中合作是隐藏还是灰显（目前灰显）。标题动效/声音开关的位置。

#### 验收
- [ ] 首次启动默认聚焦 Single Play。
- [ ] 合作显示为锁定且永不启动。
- [ ] 仅用手柄即可移动并执行菜单。
- [ ] 存在授权说明行。

#### 目的

| 输入情境 | 非战斗，焦点驱动 |
| 优先级 | 核心 |

#### 无障碍
稀有度/new 靠 颜色+边框+角标图标+文字 (§9-3)。焦点描边 ≥3:1。暂停 = 无时间压力。文字可缩放。

#### UX 设计意图
- **一眼读懂**：冻结 + 变暗 把焦点聚到三张卡；图标=类型、颜色/边框=稀有度、简短数字=效果 → 1s 读懂。
- **操作·防误触**：左侧当前构筑让你按协同选择（许多 survivor-like 缺的一环）；底部输入图符。
- **手感·爽点**：确认弹出宝石 + 短暂无敌帧 —— "我变强了"的瞬间；稀有度越高，装饰越多。
- **首次 vs 重复**：首次升级埋入防御使新手不至崩盘；老手在保底规则内快速成型。
- **无障碍**：稀有度不只靠颜色。
- **需特别注意**：① 不懂协同就选 → 构筑面板常开；② 色盲稀有度 → 颜色+边框+图标+文字；③ 7:00 重叠丢失 → 基于规则关闭再 Dawn Wake；④ 满级/类别误点 → 移出池 / 以升级形式呈现。

#### 待定问题
空池兜底类型。卡牌确认无敌帧时长。

#### 验收
- [ ] 升级立即冻结战斗并显示 3 张卡。
- [ ] 持有<6 保底 ≥1 新 + ≥1 升级；持有=6 仅升级。
- [ ] 首次升级包含 ≥1 防御物品。
- [ ] 色盲模式靠形状/文字区分稀有度。
- [ ] 仅用手柄即可聚焦并确认。

#### UX 设计意图
- **一眼读懂**：顶部大徽标以情绪传达"如何结束"；下方把 本局构筑（武器/物品）与 游玩记录（时间/击杀/闪避）左右分开，互不挤占。
- **操作·防误触**：三个按钮，默认焦点在 Retry 以便一键重玩；排行榜不可用 = 暗+原因。
- **手感·爽点**：Dream Break = 胜利的全白 + 真结局台词；Dawn Wake = 释然；失败 = 平静 —— 各分支不同情绪。新纪录 = 徽标。
- **首次 vs 重复**：失败展示 已达成点 + 下一目标，让你想再来一局；老手追逐 DB 时间/分段/无伤。
- **无障碍**：分支以文字说明；特效可切换。
- **需特别注意**：① 构筑/统计 模糊 → 分区（Brotato）；② 失败像死胡同 → 已达成/下一目标/部分奖励 (§13-3 #8)；③ 色盲分支 → 徽标文字；④ 并发结束混淆 → §4-1 优先级。

## 6. 横切规则

- **导航模型**
  - 局外（O1–O6）：焦点驱动。始终设默认焦点（主菜单=单人，角色=Kohaku，关卡=LD-001/Normal，结算=Retry，暂停=Resume）。返回/取消 = Esc/B。不可逆操作（退出/返回主菜单/解锁/开局丢失）需确认。
  - 局内（I1–I2）：实时。仅 模态（升级 I3）/ 保护（奖励 I4）/ 暂停（I5）作为覆盖层；其间战斗冻结。
  - 切换输入设备会一次性替换每个 C-PROMPT 图符（无需记忆）。
- **全局 空/加载/错误**：`loading` 区分**进入动画**与**无数据**（"加载中…/同步中…" vs 淡入）。`error` 说明**问题 + 修复**（"记录加载错误 — 重试"）；缺失值 = 上次值 + 警告边框。`empty` 绝不空白 —— 给出原因（"—"、"未发现"）。
- **无障碍（GDD §9-3）**：色盲配色（敌/友弹幕、危险区域靠 形状+颜色）、震动/闪光/后处理强度、命中判定显示、终极插画简化、自动瞄准偏置、清醒闪避辅助、字幕（大小/背景/速度）。**没有任何画面仅靠颜色传达状态。** 设置（O4）是一切的入口。
- **BM 一致性（GDD §10）**：无付费战力 / 无抽卡 → **任何画面都无现金购买 UI。** Meta（O5）仅靠游玩赚取的货币解锁。
- **布局 / 安全区**：16:9（1920×1080）。HUD 置于 边缘/顶部/底部，中央清空。左上（状态）/ 底部（资源）/ 顶部中央（Boss）槽位跨画面含义固定（P4）。21:9·4:3 在安全区内缩放（后续）。

## 附录 D. 实现绑定 — Unity 6 UI Toolkit × DOTS/ECS
> 在所选技术栈（Unity 6 · DOTS · UI Toolkit）上以**事件驱动、无 GC**方式实现 §5 数据绑定。权威参考：`unity-dots-manual`、弹幕地狱渲染配方。最终决定 = TDD。

- **角色分工**：ECS（仿真，jobs/Burst）是运行时状态的唯一真源（`RunState`、`PlayerRuntime`、Boss/试炼运行时 — §7 新增）。UI Toolkit（托管、主线程）负责渲染。一个**桥接 `SystemBase`**（主线程）读取 ECS 并**仅在变化时**推送给 UI —— 这才是"事件驱动"的真正实现。
- **无 GC / 无逐帧重建**（HUD 在 180 敌人下不可有开销，P3）：状态以 `IComponentData` 单例存在；桥接检查 ECS 的 **ChangeVersion**，只更新发生变化的标签；缓存 `VisualElement` 引用（不逐帧 `Q<>()`）；循环中无 LINQ/装箱；计时器文本按**秒**更新，而非逐帧。
- **世界空间 vs 屏幕空间**：固定 HUD = 屏幕覆盖（UIDocument/PanelSettings）；锚定玩家/敌人（清醒连击 A6、反击窗口 A12、**伤害数字**）= 世界锚点 —— **大量世界文本（伤害数字）走项目的批量渲染路径，而非 UI Toolkit**；UI Toolkit 仅用于少数锚定 HUD 的元素。
- **事件 ↔ ECS**：§5 的"UI 提议"事件以桥接的变化检测实现（对 DOTS 友好）；GDD §12-1 事件（`BossDefeated` 等）以 ECS 事件/单例标志接收，驱动 UI 转换。模态/奖励 = 仿真暂停同步（系统组停止）。
- **多分辨率 / 输入 / 本地化**：PanelSettings 缩放模式 + 安全区（追踪表 D-18）；手柄经 UI Toolkit 焦点导航（描边 ≥3:1）+ Input System；Unity Localization 字符串表（KR/EN/CN 长度，字幕 ~≤38/行，区域数字格式）。

## 附录 E. 可用性测试计划（核心循环）
> UX 设计意图是启发式推理；这里说明它如何被验证。仅基于线框图的论断（混乱下可读性）**必须**在引擎构建中重新验证。

- **目标/假设**：不迷路地通过核心循环（菜单→战斗→升级→Boss→奖励→结算）；关键可读性：① 读距 7:00 时间 ≤1s、② 在 180 敌人时识别 危险/玩家/拾取物、③ 读一张卡 ≤1s、④ 理解奖励顺序（SP→树→4 选物）。
- **方法/参与者**：有主持的出声思考，6–8 人 —— survivor-like 新手 3 + 老手 3 + 色盲 1–2（单独场次）。构建：VS 范围。
- **任务场景 ↔ 验收**：
  | # | 任务 | 成功 | 关联 |
  | --- | --- | --- | --- |
  | T1 | 从启动到开局 | ≤3 分钟，无引导，选角色+关卡→开始 | O1/O2/O3 |
  | T2 | （随机暂停）"距 7:00 还有多久？" | ≤1s 答对 | I1 A2 (§3-2) |
  | T3 | 选一张升级卡 | 读 ≤1s，**观察是否使用构筑面板** | I3 B3/B4 |
  | T4 | 尝试清醒闪避 | 新手误触 + 看到连击 / 老手有意 | I1 A6 (§3-2) |
  | T5 | Boss 战 | 看到 Boss 条，裂痕→眩晕 与 反击 被**使用** | I2 A10/A11/A12 |
  | T6 | Boss 奖励 | **理解 SP→树→4 选物 顺序**，节点/物品 可区分 | I4 |
  | T7 | 结算 | 识别分支、构筑 vs 统计 分区、Retry 重开 | I6 |
- **指标**：逐任务 成功/时间/错误 · SUS · 乐趣/投入度问卷 · **混乱下可读性**（冻结的高密度截图识别准确率，色盲场次单独）· 无障碍（色盲识别 / 开关行为）。
- **通过线 / 迭代**：核心任务（T1·T2·T3·T6）成功率 ≥85%，T2 的 1s 读取 ≥90%，色盲致命物识别 =100%。未达标 → 在追踪表调整对应数值并复测；并在 §8 反映。
- **局限**：这是*测试设计*。真实可点击原型 / 场次 / SUS 统计 需要构建 + 人员 —— 此处未执行。
